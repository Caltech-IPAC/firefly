Searching the Image Service

There are five main options for searching the Image Service. Searching by position is the most commonly-used option.

Options Common to Most Searches

The individual search options are given below, in the same order in which they appear in the search pane. In any case, most of the search panes have the following additional options:
Each of the "Search" panes ask for the "Return Image size". Returned images can be up to 2 degrees (or the equivalent in arcsec and arcmin) on a side. After you search the first time and are presented with the results page, simply click the "Searches" tab to subsequently modify your search.

Position Search

This search is the most common search performed on astronomical archives. Enter your central position, and all observations meeting the rest of your entered criteria are returned.

You may enter a target name, and have either NED or SIMBAD resolve the target name into coordinates. Alternatively, you may enter coordinates directly. These coordinates can be in decimal degrees, in hh:mm:ss dd:mm:ss, hh mm ss dd mm ss, or hhhmmmsss dddmmmsss format.

The position search can also be executed in "batch mode" for a list of objects given in a file (select the Multi-Object tab). The file must be in IPAC table format, which is basically ASCII text with headers explaining the type of data in each column, separated by vertical bars. A sample input file looks like this:

Please use the IPAC table check tool to check and reformat your input table before the table upload. Also pay attention to the following if the table upload search still does not work properly:

1. No hyphen allowed in column names.
2. Column names should be all in lower case
3. No empty lines at the end of the input file.

For both single- and multi-object searches, you also need to select the Image Set to search (Preliminary or All-Sky release), as well as the desired product level: coadded Atlas images, Single Exposure (Level 1b) frames, or both. Note that if you are interested in retrieving all the single exposure frames (or atlas tiles) that contain your target, you will need to set "Return only the most centered image containing the target" to "no".

Solar System Object/Orbit (all-sky)

It is also possible to search for specific solar system objects in the single-frame WISE images. The search can be performed for comets or asteroids using the object name or the six orbital parameters supplied by the user either individually or in the Minor Planet Center (MPC) one-line element format.

Solar System Object/Orbit Search Types:

    Object Name: Search by object names by entering the name in the "Object Name" field. The accepted naming conventions are the ones that have been approved for use by the CSBN and its predecessors. News reports occasionally use unofficial or unapproved names: you cannot use these names to access objects. For asteroid names, one can enter either the ID number, name or designation, e.g. 2, 887, 1917, Pallas, Alinda, Cuyo, 1981 QB, 1996 GQ, or 2010 CG18. It can also handle names with apostrophes and dashes like O'Connell and Pic-du-Midi. For comet names, one can enter either the whole name e.g. 10P/Tempel 2, 73P-B/Schwassmann-Wachmann 3, P/2009 WJ50, or P/2010 D1 or just the unique part of its name e.g. 10P, 73P-B, 2009 WJ50, or 2010 D1.

    The planets and their satellites are not implemented in this version. However, Pluto is considered to be a minor planet according to the IAU. It can be searched using its name "Pluto" or its number "134340".

    MPC Input: Search for objects using a properly-formatted Minor Planet Center input string. This option allows the user to cut-and-paste a line directly from a table of orbital elements in MPC Format into the search field. There are a number of orbital element tables available at the MPC website, for example, observable NEO and observable comets. The complete list of minor planets can be found at the MPC Orbit (MPCORB) Database.

    Information on the format of the element tables is given by following sites: minor planet format and comet format.

    where e=eccentricity, a=semimajor axis, M=mean anomaly, i=inclination, Node=longitude of the ascending node, w=argument of perihelion, q=perihelion distance, Tp=time of perihelion passage, n=mean motion, H=absolute magnitude and G=slope parameter.

    Manual Input: The standard six orbital elements for asteroid are eccentricity, semimajor axis, mean anomaly, inclination, longitude of the ascending node, and argument of perihelion. For comets, the elements are eccentricity, perihelion distance, time of perihelion passage, inclination, longitude of the ascending node, and argument of perihelion. Notice that for comets, perihelion distance is used instead of semimajor axis and time of perihelion passage is used instead of mean anomaly. The reason for replacing these two parameters is to allow for cases where long period comets have a parabolic or hyperbolic orbit.

    A good description of the orbital parameters are given in JPL's Solar System Dynamics web site.

Additional constraints are also allowed:

    Observation Begin/End (UT): Allows the search to be constrained to a given time range.

    Return Image Size: Specifies the size of the image to be returned. The pull-down menu is used to select the units in which to express this size. Leave the field blank to obtain the WISE full (0.78x0.78 deg) images.

    Return the following bands: Outputs can be restricted to any or all of the four WISE bands.
