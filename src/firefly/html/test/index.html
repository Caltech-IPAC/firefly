How to write your own tests

To add a test to an existing page

Each test is presented as a template. It consists of 3 parts: expected: the expected results. This can be a description, an image, or both actual: the rendered results of executing the script script: script to run. Try to keep it clean and easy to understand. Below is a complete example of an actual test: <template title="GAIA tap search" class="tpl sm"> <div id="expected" class="source-code indent-2"> Table successfully loaded - VOTable - first 3 rows of gaia_source </div> <div id="actual" class="box x3"></div> <script> onFireflyLoaded = function (firefly) { tblReq = firefly.util.table.makeTblRequest('AsyncTapQuery', 'GAIA tap search', { serviceUrl: "http://gea.esac.esa.int/tap-server/tap", QUERY: "SELECT TOP 3 * FROM gaiadr2.gaia_source", }); firefly.showTable('actual', tblReq); } </script> </template> You may noticed CSS styles were used. For your convenience, there are predefined styles in the included styles.css file. These styles are designed for composition, meaning they were made to be used together. For instance, to make a horizontal flex box three times the size of regular box. <div class="flow-h box x3" /> Below is a list of other styles you can use: ( look in styles.css for a complete listing ) flow-h :horizontal flex box flow-v :vertical flex box tpl :define a standard template block + xs :extra small template + sm :small template + med :medium template + lg :large template + xl :extra-large template box :a standard square box to display content + xs :extra small; 1/2 size of a regular box + sm :small; 2/3 size of a regular box + x2 :2 times the size of a regular box + x3 :3 times the size of a regular box + x4 :4 times the size of a regular box source-code :format as source code + indent-1 :source need to adjust for 1 indentation + indent-2 :source need to adjust for 2 indentation + indent-3 :source need to adjust for 3 indentation + indent-4 :source need to adjust for 4 indentation + indent-5 :source need to adjust for 5 indentation expected-list :turn on normal list styling, use with ul element (firefly turns it off) exclusive :if defined on any template, display only that template (for debugging)

To add a new test page

Creating a new test page is pretty easy. Create a blank html file like below. Make sure you include styles.css and template_loader.js into the head section. After that, start adding test into your page in the form of a template. Make sure to include your new page into this index.html file so it can be easily accessed. <html> <head> <title>Title of your test page</title> <link rel="stylesheet" href="styles.css"> <script type="text/javascript" src="template_loader.js"></script> </head> <body> <!-- include your test template(s) here --> <!-- this is where test cases will be attached--> <div id="tst-container"/> </body> </html>
